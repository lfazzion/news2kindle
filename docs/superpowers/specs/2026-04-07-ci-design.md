# CI Pipeline Design

**Date:** 2026-04-07

## Overview

Add a CI workflow that runs on every push and pull request to catch lint errors and test failures before code reaches main.

## Triggers

- `push` — all branches
- `pull_request` — all branches

## Workflow: `.github/workflows/ci.yml`

Single job `ci` running on `ubuntu-latest` with Python 3.13.

### Steps

1. `actions/checkout@v4`
2. `actions/setup-python@v5` with `cache: pip` and `python-version: '3.13'`
3. `pip install -r requirements.txt`
4. `ruff check .` — fails on lint errors
5. `ruff format --check .` — fails if files are not formatted
6. `python -m pytest tests/ -v` — fails if any test breaks

### Environment Variables

None required. `tests/conftest.py` already injects dummy env vars before imports, so no secrets are needed for the test run.

## Out of Scope

- Coverage reports / artifacts
- Matrix builds (multiple Python versions)
- Parallel lint + test jobs
